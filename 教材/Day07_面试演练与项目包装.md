# Day 7：面试演练与项目包装（把它变成“拿得出手的作品”）

目标：你要能做到 3 件事：

1) 30 秒说清楚项目
2) 2 分钟讲清楚架构与亮点
3) 面试追问能顶住（尤其是质量保障、稳定性、可扩展性）

---

## 1. 30 秒版本（背下来）

> 这是一个 AI 用例生成平台。用户上传 PRD 后，后端解析文档并可结合知识库做 RAG 检索增强，然后以异步批次任务方式调用大模型生成测试点和测试用例，结果结构化落库并支持导出 Excel。前端通过轮询批次状态展示进度，整体支持本地开发与 Docker Compose 部署。

### 1.1 手把手：把 30 秒讲稿练到“张嘴就来”

新手最有效的方法是“录音复盘”：

1) 打开手机录音
2) 读一遍上面 30 秒稿
3) 复听：是否有停顿、是否有口头禅
4) 把卡壳的地方用更短的词替换
5) 重复 3 遍

你不需要把自己讲得多华丽，只要“清晰、不卡、逻辑顺”。

## 2. 2 分钟版本（结构化讲法）

按“痛点→方案→架构→亮点→验证”讲：

### 2.1 痛点

- 需求文档长，人工提炼测试点与用例成本高
- 大模型如果不约束，输出不稳定且容易幻觉

### 2.2 方案

- 文档解析 + 知识库（RAG）提供依据
- Prompt 约束结构化 JSON 输出
- 异步批次任务提高稳定性与体验

### 2.3 架构（你可以指文件讲）

- 后端入口：`backend/app/main.py`
- API 聚合：`backend/app/api/v1/router.py`
- AI 操作：`backend/app/api/v1/ai_operations.py`
- 任务实现：`backend/app/services/ai/test_point_extractor.py`、`test_case_generator.py`
- RAG：`backend/app/services/rag_service.py`
- 前端轮询：`frontend/src/components/AiProgressModal.vue`

### 2.4 亮点（面试官最爱听的）

- 批次任务状态落库，可观测、可恢复
- RAG 降低幻觉，提高结果贴合业务
- Prompt 集中管理，便于调优
- Docker Compose + Nginx 反代，交付可运行

### 2.5 验证

- 演示一条闭环：上传 → 提取测试点 → 生成用例 → 导出
- 展示批次进度/失败信息（如果你做了 Day 6 改动会更强）

## 3. 演示脚本（你照着点，不会乱）

建议 3 分钟演示：

1) 打开项目列表 → 新建项目
2) 上传一份 PRD（提前准备小文件，避免演示卡住）
3) 点击“提取测试点” → 弹出进度 → 完成后展示测试点
4) 选择测试点 → 点击“生成测试用例” → 完成后展示用例
5) 点击“导出 Excel”

### 3.1 手把手：演示前你必须做的准备（避免现场翻车）

你在面试前一天做这 5 件事：

1) 准备一份“小 PRD 文档”（不要太大，1-2 页最好）
2) 确保你的 Key 与模型配置已生效（用 `GET /settings` 验证）
3) 确保你能 100% 跑通一次生成（至少成功一次）
4) 把浏览器缓存清掉或用无痕窗口（避免旧数据干扰）
5) 演示当天只跑本地，不要现场拉依赖（npm install 很慢）

演示时你要边讲边强调：

- 生成是异步批次，不会卡住请求
- 批次状态可查询，失败有错误信息

## 4. 追问题库（你至少准备这些）

### 4.1 “你怎么保证生成质量？”

- 用 RAG 提供依据
- Prompt 强约束输出 JSON
- 解析失败重试/校验字段

### 4.2 “为什么用轮询，不用 WebSocket？”

- 原型期轮询实现简单稳定
- 任务频率不高时开销可控
- 规模上来可以演进到 SSE/WebSocket

### 4.3 “你们怎么做测试/CI？”（项目现状短板）

你要诚实但给出方案：

- 核心 service 层写单测（mock LLM 返回固定 JSON）
- 写 API 集成测试（起 test DB）
- 在 CI 里跑 lint/typecheck/test

#### 手把手：你怎么把“短板”讲成“加分点”

你可以这样回答（模板）：

> “当前版本更偏原型交付，自动化测试与 CI 还没体系化，这是我会补齐的方向。我会优先从 service 层做单测，用 mock LLM 固定返回保证可重复，然后补 API 集成测试，最后把 lint/typecheck/test 接到 CI，确保每次改动都可验证。”

这段回答的关键是：

- 承认现状
- 给出有顺序、有取舍的落地方案

### 4.4 “任务如果服务重启会怎样？”

- 当前 `asyncio.create_task` 可能丢任务（视实现而定）
- 生产化方案：引入任务队列（Celery/RQ/Arq）+ 可重试 + 持久化任务状态

### 4.5 “数据与隐私怎么处理？”

- Key 不落日志、前端脱敏显示
- 文档与知识库可删除
- 生产需加鉴权、审计、加密存储
