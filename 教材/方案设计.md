# AI测试用例生成平台 - 方案设计

## 1. 项目概述

构建一个AI驱动的测试用例生成平台，核心流程：读取PRD文档 → AI提取测试点 → 生成可执行测试用例。支持 RAG 知识库增强生成质量、飞书通知、设置持久化。

---

## 2. 技术栈

| 层级 | 选型 | 理由 |
|------|------|------|
| 后端框架 | **FastAPI** | 异步原生，自动OpenAPI文档，Pydantic校验 |
| ORM | **SQLAlchemy 2.0 (async)** | 成熟稳定，支持异步 |
| 数据库 | **SQLite (aiosqlite)** | 零配置，开箱即用，适合单机部署 |
| LLM集成 | **LiteLLM** | 统一接口调用OpenAI/Claude/DeepSeek等100+模型 |
| 向量数据库 | **ChromaDB** | 文件持久化，无需额外服务，适合 RAG 场景 |
| Embedding | **sentence-transformers** | 本地运行 `all-MiniLM-L6-v2`，免费无需API；可切换 OpenAI embedding |
| PDF解析 | **PyMuPDF (pymupdf4llm)** | 速度快，输出LLM友好的Markdown |
| DOCX解析 | **python-docx** | 直接访问Word对象模型 |
| HTTP客户端 | **httpx** (async) | 飞书webhook和外部调用 |
| 前端框架 | **Vue 3 + Vite + TypeScript** | 轻量高效，适合工具型UI |
| UI组件库 | **Element Plus** | 组件丰富，表格/表单能力强，界面全中文 |
| 状态管理 | **Pinia** | Vue 3 官方状态管理方案 |

## 6. 核心模块设计

### 6.1 LLM客户端 (`backend/app/services/ai/llm_client.py`)

基于LiteLLM封装的统一LLM调用客户端：
- 通过模型字符串切换提供商（如 `openai/gpt-4o`、`anthropic/claude-sonnet-4-20250514`）
- 支持自定义 API Base URL（代理/第三方平台如 SiliconFlow）
- 支持JSON结构化输出模式
- 内置指数退避重试（默认3次）
- 自动记录token用量

### 6.2 文档解析器 (`backend/app/services/ai/document_parser.py`)

根据文件类型分发到对应解析器：
- **PDF** → pymupdf4llm 转 Markdown（保留标题层级和表格）
- **DOCX** → python-docx 提取段落和表格，转 Markdown
- **MD** → 直接读取原始内容

输出 `ParseResult`：包含 `raw_text`、`markdown`、`metadata`（页数、标题等）

### 6.3 测试点提取 (`backend/app/services/ai/test_point_extractor.py`)

PRD → 测试点的核心管道：
1. 创建 GenerationBatch 记录（状态 running）
2. 后台异步执行（`asyncio.create_task`），立即返回 batch_id
3. 合并多个文档的Markdown内容
4. 若选择了知识库，调用 RAG 检索相关片段作为补充上下文
5. 若为重新生成，附带已有测试点和用户反馈
6. 从 `prompts.py` 构建系统提示词+用户提示词
7. 调用LLM，要求JSON对象格式输出
8. 解析为结构化测试点列表，跳过空测试点
9. 更新 batch 状态为 completed / failed

LLM输出格式：
```json
{
  "test_points": [{
    "title": "测试点标题",
    "description": "详细描述",
    "priority": "P0",
    "category": "functional",
    "preconditions": "前置条件",
    "expected_result": "预期结果",
    "source_context": "来源PRD片段"
  }]
}
```

### 6.4 测试用例生成 (`backend/app/services/ai/test_case_generator.py`)

测试点 → 可执行测试用例：
- 按测试点逐个生成，每个测试点可生成多条用例（正向/反向/边界）
- 后台异步执行，支持 RAG 知识库增强

LLM输出格式：
```json
{
  "test_cases": [{
    "title": "用例标题",
    "preconditions": "前置条件",
    "steps": [
      {"step_number": 1, "action": "操作步骤", "expected_result": "预期结果"}
    ],
    "priority": "P1",
    "case_type": "positive"
  }]
}
```

### 6.5 RAG 知识库 (`backend/app/services/rag_service.py`)

基于 ChromaDB 的 RAG 检索增强生成：
- **文本分块**：按字符分块（可配置 chunk_size 和 overlap）
- **向量索引**：每个知识库对应一个 ChromaDB collection（`kb_{id}`）
- **Embedding**：支持本地 sentence-transformers（默认）和 OpenAI embedding
- **检索**：多知识库联合检索，返回 top_k 最相关片段
- **上下文注入**：检索结果格式化后注入 LLM prompt

### 6.6 配置持久化 (`backend/app/config.py`)

- 支持 `.env` 文件 + 环境变量 + `settings.json` 三层配置
- `settings.json` 存储用户通过 UI 修改的配置（API Key、模型、Webhook等）
- `settings.json` 优先于环境变量，确保 UI 设置始终生效
- API Key 在接口返回时脱敏显示（保留前6位和后4位）

### 6.7 飞书通知 (`backend/app/integrations/feishu.py`)

通过Webhook发送飞书消息：
- 支持纯文本和富文本卡片消息
- 可配置HMAC-SHA256签名验证
- 在测试点提取和用例生成完成后自动触发
- 消息包含：项目名称、生成数量、平台链接

## 7. 用户操作流程

```
┌──────────────────────────────────────────────────────────────┐
│  1. 创建项目                                                   │
│     ↓                                                          │
│  2. 上传PRD文档（PDF/DOCX/MD，支持多文件）                        │
│     ↓  （自动解析为Markdown文本）                                 │
│  3. [可选] 创建知识库，上传参考资料（接口文档、历史用例、业务规则等）   │
│     ↓  （自动分块 → Embedding → 存入向量数据库）                   │
│  4. 选择文档 → 点击"AI提取测试点" → 可选关联知识库 → AI生成测试点    │
│     ↓  （后台异步执行，可关闭弹窗继续操作；飞书通知：测试点已生成）    │
│  5. 查看/编辑/删除/新增测试点（人工干预）                           │
│     ↓                                                          │
│  ┌─ 不满意？→ 追加文档 + 输入反馈 → "重新生成" → 回到步骤5          │
│  │                                                              │
│  └─ 满意 → 6. 选择测试点 → "一键生成用例" → 可选关联知识库           │
│               ↓  （后台异步执行；飞书通知：测试用例已生成）            │
│            7. 查看/编辑测试用例                                    │
│               ↓                                                  │
│            8. 导出Excel                                          │
└──────────────────────────────────────────────────────────────┘
```

## 8. 前端页面设计

### 8.1 项目列表页 (`/`)
- 卡片或表格展示所有项目
- 显示每个项目的文档数、测试点数、用例数统计
- "新建项目"按钮

### 8.2 项目详情页 (`/projects/:id`)
- 标签页切换：**文档** | **测试点** | **知识库** | **测试用例**
- **文档标签页**：文档上传，文档列表（含解析状态徽标），"AI提取测试点"按钮
- **测试点标签页**：可编辑表格，支持筛选排序、批量操作，状态标签（已就绪/生成中）。工具栏含"手动新增"、"一键生成用例"、"批量删除"
- **知识库标签页**：折叠面板展示知识库列表，每个知识库可上传/查看/删除文档，显示文档数和分块数
- **测试用例标签页**：可展开表格（展开查看步骤），导出按钮，状态徽标（草稿/评审/通过/生成中）

### 8.3 AI配置对话框
- 点击"AI提取测试点"或"一键生成用例"时弹出
- 可选关联知识库（多选下拉框）
- 确认后启动后台生成

### 8.4 设置页 (`/settings`)
- LLM配置：API Base URL、默认模型（可选预设或自定义输入）、OpenAI API Key、Anthropic API Key
- 飞书通知配置：Webhook URL、签名密钥、测试连接按钮
- 设置自动持久化，重启后保留

### 8.5 AI进度弹窗 (`AiProgressModal`)
- AI操作时显示加载动画和状态文本
- 支持关闭弹窗在后台继续运行
- 轮询 `GET /ai/batches/{batch_id}` 查询进度
- 完成/失败后自动刷新数据并提示
- 切换项目后返回时自动恢复运行中批次的状态

## 10. 验证方式

1. **后端API**：通过 FastAPI 自动文档 (`/docs`) 手动测试所有接口
2. **文档解析**：上传 PDF/DOCX/MD 文件，验证解析输出质量
3. **知识库**：创建知识库，上传文档，确认状态变为 completed、chunk_count > 0
4. **AI提取**：上传真实PRD，验证测试点提取的质量和完整性
5. **RAG增强**：对比有/无知识库生成的测试点差异，验证覆盖度提升
6. **用例生成**：基于测试点生成用例，验证步骤完整性和可执行性
7. **飞书通知**：配置测试webhook，验证消息发送和内容格式
8. **设置持久化**：修改设置后重启服务，验证配置保留
9. **全流程E2E**：创建项目 → 上传文档 → 创建知识库 → 提取测试点（勾选知识库） → 生成用例 → 导出Excel
